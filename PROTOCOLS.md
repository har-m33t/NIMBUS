## 1. WebSocket Message Schema

All messages traverse a single AWS API Gateway WebSocket channel. Every payload MUST be valid JSON and MUST include the `sessionId` field.

### 1.1 — Client → Backend (Inference Request)

Sent by the `frontend` local app at the agreed keypoint frequency (see §3.1).

```json
{
  "action": "INFER",
  "sessionId": "<uuid-v4>",
  "roomId": "<zoom-meeting-id>",
  "timestamp": "<ISO-8601-UTC>",
  "sequenceNumber": 1024,
  "payload": {
    "keypoints": {
      "leftHand": [
        { "x": 0.512, "y": 0.341, "z": -0.042 }
      ],
      "rightHand": [
        { "x": 0.488, "y": 0.298, "z": -0.031 }
      ],
      "pose": [
        { "x": 0.500, "y": 0.700, "z": 0.000, "visibility": 0.99 }
      ]
    },
    "includeFaceCrop": false
  }
}
```

**Field Rules:**
| Field | Type | Required | Notes |
|---|---|---|---|
| `action` | `string` | ✅ | Always `"INFER"` for keypoint frames |
| `sessionId` | `string` (UUID v4) | ✅ | Tracks user across Lambda + DynamoDB |
| `roomId` | `string` | ✅ | Zoom meeting ID for caption routing |
| `timestamp` | `string` (ISO-8601) | ✅ | UTC time at moment of frame extraction |
| `sequenceNumber` | `integer` | ✅ | Monotonically increasing; used for ordering |
| `keypoints.leftHand` | `array[21]` | ✅ | 21 MediaPipe hand landmarks, normalized 0.0–1.0 |
| `keypoints.rightHand` | `array[21]` | ✅ | 21 MediaPipe hand landmarks, normalized 0.0–1.0 |
| `keypoints.pose` | `array[33]` | ✅ | 33 MediaPipe pose landmarks, normalized 0.0–1.0 |
| `includeFaceCrop` | `boolean` | ✅ | Set `true` once every 10 frames (see §3.2) |

> **Coordinate System:** All `x`, `y`, `z` values MUST be MediaPipe **normalized** floats in the range `[0.0, 1.0]`. Absolute pixel values are prohibited.

---

### 1.2 — Backend → Client (Inference Results)

The backend may emit multiple events per single inference request. The client MUST handle all `type` values defined below.

#### `type: "GLOSS"` — Intermediate ASL Tokens
Emitted immediately after SageMaker returns raw gloss tokens. Used for real-time UI feedback.

```json
{
  "type": "GLOSS",
  "sessionId": "<uuid-v4>",
  "timestamp": "<ISO-8601-UTC>",
  "sequenceNumber": 1024,
  "payload": {
    "tokens": ["STORE", "I", "GO"],
    "confidence": 0.87
  }
}
```

#### `type: "EMOTION"` — Detected Facial Emotion
Emitted after Rekognition processes a face crop frame.

```json
{
  "type": "EMOTION",
  "sessionId": "<uuid-v4>",
  "timestamp": "<ISO-8601-UTC>",
  "payload": {
    "emotion": "HAPPY",
    "confidence": 0.93,
    "allEmotions": {
      "HAPPY": 0.93,
      "CALM": 0.05,
      "SAD": 0.02
    }
  }
}
```

Allowed `emotion` values: `HAPPY | SAD | ANGRY | CALM | SURPRISED | FEAR | DISGUSTED | CONFUSED`

#### `type: "CAPTION"` — Final Natural English Sentence
Emitted after Bedrock completes sentence interpretation. This is the final output pushed to Zoom and Polly.

```json
{
  "type": "CAPTION",
  "sessionId": "<uuid-v4>",
  "timestamp": "<ISO-8601-UTC>",
  "sequenceNumber": 1024,
  "payload": {
    "text": "I am going to the store.",
    "emotion": "HAPPY",
    "audioUrl": "s3://nimbus-prod-tts-audio/<sessionId>/<uuid>.mp3",
    "latencyMs": 340
  }
}
```

#### `type: "ERROR"` — Pipeline Failure
Emitted on any service failure (see §4 for fallback values).

```json
{
  "type": "ERROR",
  "sessionId": "<uuid-v4>",
  "timestamp": "<ISO-8601-UTC>",
  "payload": {
    "code": "SAGEMAKER_INFERENCE_FAILED",
    "glossFallback": "[UNKNOWN_SIGN]",
    "message": "SageMaker endpoint returned a non-200 response."
  }
}
```

#### `type: "SIGNAL"` — Signaling Events (Room Management)

```json
{
  "type": "SIGNAL",
  "event": "JOIN_ROOM | LEAVE_ROOM | ICE_CANDIDATE | NEW_CAPTION",
  "sessionId": "<uuid-v4>",
  "roomId": "<zoom-meeting-id>",
  "payload": {}
}
```

Allowed `event` values: `JOIN_ROOM | LEAVE_ROOM | ICE_CANDIDATE | NEW_CAPTION`

## 2. Global State & Context Persistence

### 2.1 — Session Lifecycle

Each user session is identified by a `sessionId` (UUID v4) generated by the **client** at application start. The `roomId` maps to the active Zoom meeting. Both are passed in every message.

DynamoDB table: `NIMBUS_PROD_Sessions`

```
PK: sessionId (string)
SK: timestamp (ISO-8601)
Attributes:
  roomId          (string)
  glossBuffer     (list<string>)  — rolling window of raw tokens
  lastEmotion     (string)        — most recently detected emotion label
  lastCaptionAt   (timestamp)     — when Bedrock last flushed a sentence
  connectionId    (string)        — API Gateway WebSocket connection ID
  ttl             (number)        — Unix epoch, session auto-expires after 4 hours
```

### 2.2 — Gloss Buffer & Sentence Boundary

Bedrock is NOT called on every keypoint frame. The Lambda accumulates gloss tokens in `glossBuffer` in DynamoDB and flushes only when a sentence boundary is detected.

**Sentence Boundary Rules (in priority order):**

| Rule | Trigger | Action |
|---|---|---|
| **Token Count** | Buffer reaches **15 tokens** | Force-flush to Bedrock |
| **Time Window** | **3 seconds** elapsed since first token in buffer | Flush to Bedrock |
| **Pause Signal** | No new `INFER` messages for **1.5 seconds** | Flush to Bedrock |
| **Explicit Boundary Token** | SageMaker emits `[EOS]` token | Flush immediately |

The Lambda passes the **full buffer contents + last detected emotion** to Bedrock as context in a single prompt invocation.

### 2.3 — Context Window for Bedrock

To enable Bedrock to be context-aware across sentences, the Lambda retrieves the **last 3 completed captions** from DynamoDB and includes them in the prompt as prior conversation context. This enables pronoun resolution, topic continuity, and more natural translations.

## 3. Media Processing Pipeline Rules

### 3.1 — Keypoint Extraction Frequency

| Parameter | Value | Rationale |
|---|---|---|
| **Keypoint Send Rate** | **10 FPS** | Sufficient for ASL motion resolution; limits WebSocket traffic |
| **Minimum Frame Skip** | 100ms between sends | Prevents burst floods |
| **Max Batch Size** | 1 frame per WebSocket message | No batching; latency-critical path |

### 3.2 — Face Crop Frequency for Rekognition

| Parameter | Value |
|---|---|
| **Face crop extraction rate** | Once every **10 keypoint frames** (≈ every 1 second at 10 FPS) |
| **Trigger field** | `includeFaceCrop: true` in the `INFER` payload |
| **Face image format** | JPEG, base64-encoded, max **640×480px** |
| **Face image field** | `payload.faceCropBase64` (only present when `includeFaceCrop: true`) |

When `includeFaceCrop: true`, the full payload has an additional field:

```json
{
  "payload": {
    "keypoints": { ... },
    "includeFaceCrop": true,
    "faceCropBase64": "<base64-encoded-JPEG-string>"
  }
}
```

### 3.3 — Coordinate System (Mandatory)

- **System:** MediaPipe normalized coordinates
- **Range:** All `x`, `y` values `∈ [0.0, 1.0]` where `(0,0)` is **top-left**
- **Depth:** `z` values are relative depth (MediaPipe convention); NOT absolute
- **Visibility:** Only `pose` landmarks include a `visibility` float `∈ [0.0, 1.0]`
- **Absent hands:** If a hand is not detected, send an empty array `[]` for that hand

## 4. Error Handling & Fallback States

### 4.1 — Service Failure Fallbacks

| Failing Service | Error Code | Fallback Behavior | UI Display |
|---|---|---|---|
| SageMaker (ASL) | `SAGEMAKER_INFERENCE_FAILED` | Emit `glossFallback: "[UNKNOWN_SIGN]"` | Show `[UNKNOWN_SIGN]` in gloss ticker |
| Amazon Bedrock | `BEDROCK_TIMEOUT` | Return raw gloss tokens as the caption | Show raw gloss string in caption bar |
| Amazon Rekognition | `REKOGNITION_FAILED` | Default emotion to `"CALM"` | No UI alert; silent fallback |
| Amazon Polly | `POLLY_FAILED` | Captions still delivered; skip audio | Show caption; audio indicator turns gray |
| WebSocket Disconnect | `WS_DISCONNECT` | Client retries with exponential backoff (max 5 retries, starting at 500ms) | Show "Reconnecting…" banner |

### 4.2 — Latency Thresholds & Signal Indicator

The NIMBUS UI must display a real-time signal quality indicator driven by end-to-end pipeline latency, measured from `INFER` send time to `CAPTION` receive time.

| Latency | Signal Status | UI Color |
|---|---|---|
| `< 800ms` | **Strong** | 🟢 Green |
| `800ms – 1500ms` | **Degraded** | 🟡 Yellow |
| `> 1500ms` | **Poor** | 🔴 Red |
| No response in `5000ms` | **Offline** | ⚫ Gray |

Latency is computed by the client using:
```
latency = Date.now() - message.sequenceTimestamp
```

### 4.3 — SageMaker Cold Start Handling

SageMaker endpoints on `ml.g5.xlarge` may have cold start delays of 30–90 seconds after idle periods. The Lambda MUST:

1. Ping the endpoint with a `DescribeEndpoint` call before the session begins.
2. If endpoint status is not `InService`, emit a `SIGNAL` event with `event: "ENDPOINT_WARMING"` to the client.
3. The client shows a "Warming up AI model…" loading state.

## 5. Resource Naming Conventions

All AWS resources MUST follow the `NIMBUS_PROD_` prefix. No resource may be created without this prefix.

### 5.1 — S3 Buckets

| Bucket Purpose | Name |
|---|---|
| SageMaker model artifacts | `nimbus-prod-model-artifacts` |
| Polly TTS audio output | `nimbus-prod-tts-audio` |
| SSML Prosody config (see §6) | `nimbus-prod-config` |
| User session data exports | `nimbus-prod-session-exports` |

> S3 bucket names are globally unique and must be lowercase with hyphens.

### 5.2 — Lambda Functions

| Function Name | Responsibility |
|---|---|
| `NIMBUS_PROD_WS_Connect` | WebSocket `$connect` route handler |
| `NIMBUS_PROD_WS_Disconnect` | WebSocket `$disconnect` route handler |
| `NIMBUS_PROD_ProcessFrame` | Main orchestration: SageMaker → Bedrock → Rekognition → Polly |
| `NIMBUS_PROD_PushCaption` | Pushes final caption text to the Zoom Closed Caption API |
| `NIMBUS_PROD_WarmEndpoint` | Scheduled keep-warm ping to SageMaker endpoint |

### 5.3 — DynamoDB Tables

| Table Name | Purpose |
|---|---|
| `NIMBUS_PROD_Sessions` | Active session state and gloss buffers |
| `NIMBUS_PROD_Vocabulary` | ASL vocabulary reference and gloss metadata |

### 5.4 — SageMaker

| Resource | Name |
|---|---|
| Training Job prefix | `nimbus-prod-asl-train-<YYYYMMDD>` |
| Model name | `nimbus-prod-asl-transformer-v<N>` |
| Endpoint Config | `nimbus-prod-asl-endpoint-config-v<N>` |
| Endpoint name | `nimbus-prod-asl-endpoint` |

### 5.5 — IAM Roles

| Role Name | Grants Access To |
|---|---|
| `NIMBUS_PROD_LambdaExecutionRole` | SageMaker, Bedrock, Rekognition, Polly, DynamoDB, S3, CloudWatch |
| `NIMBUS_PROD_SageMakerTrainingRole` | S3 (model artifacts bucket), CloudWatch |

> All roles use the **principle of least privilege**. The Lambda role does NOT have admin access.

### 5.6 — CloudWatch Log Groups

All Lambda functions MUST log to `/aws/lambda/<FunctionName>` with a **14-day retention policy**.

## 6. SSML Prosody Mapping

### 6.1 — Decision: Config lives in S3

The SSML prosody map is stored as a JSON config file in S3 at:

```
s3://nimbus-prod-config/ssml_prosody_map.json
```

**Rationale:** The Lambda (`NIMBUS_PROD_ProcessFrame`) reads this file once per cold start and caches it in memory. This means emotion-to-prosody mappings can be tuned by editing a single file in S3 with zero Lambda redeployment.

### 6.2 — `ssml_prosody_map.json` Schema

```json
{
  "version": "1.0",
  "defaultVoiceId": "Matthew",
  "mappings": {
    "HAPPY":     { "pitch": "+15%", "rate": "110%", "volume": "loud" },
    "SAD":       { "pitch": "-10%", "rate": "85%",  "volume": "soft" },
    "ANGRY":     { "pitch": "+20%", "rate": "120%", "volume": "x-loud" },
    "CALM":      { "pitch": "+0%",  "rate": "95%",  "volume": "medium" },
    "SURPRISED": { "pitch": "+25%", "rate": "115%", "volume": "loud" },
    "FEAR":      { "pitch": "+10%", "rate": "130%", "volume": "soft" },
    "DISGUSTED": { "pitch": "-5%",  "rate": "90%",  "volume": "medium" },
    "CONFUSED":  { "pitch": "+5%",  "rate": "95%",  "volume": "medium" }
  }
}
```

### 6.3 — SSML Template

The Lambda builds the final SSML string using the map above:

```xml
<speak>
  <prosody pitch="{pitch}" rate="{rate}" volume="{volume}">
    {captionText}
  </prosody>
</speak>
```

## 7. Responsibility & Contract Summary

| Component | Owner | Input Contract | Output Contract |
|---|---|---|---|
| **WebSocket Signaling / Room SFU** | Member 1 (Infrastructure) | Client connects with `sessionId` + `roomId` | Routes `JOIN_ROOM`, `ICE_CANDIDATE`, `LEAVE_ROOM`, `NEW_CAPTION` signal events |
| **AI Orchestration** | Member 2 (Backend) | `INFER` payload with keypoints + optional face crop | `GLOSS`, `EMOTION`, `CAPTION`, `ERROR` events back to client |
| **ML Inference (SageMaker)** | Member 3 (ML) | Flat array of normalized keypoint floats shaped `(1, T, 258)` | Array of gloss token strings + confidence score |
| **Bedrock Prompt** | Member 2 | Gloss token array + emotion label + last 3 captions (context) | Single grammatically correct English sentence string |
| **SSML / Polly** | Member 2 | English sentence + emotion label → reads `ssml_prosody_map.json` from S3 | Pre-signed S3 URL to MP3 audio file |
| **Zoom Caption Push** | Member 1 | Final `CAPTION` text string + `roomId` | HTTP POST to Zoom `/closedcaption` endpoint |

### SageMaker Input Shape Contract

The ML model (`NIMBUS_PROD_SageMakerTrainingRole`) accepts a **single inference request tensor** of shape:

```
(batch=1, time_steps=T, features=258)
```

Where `features=258` is composed as:

| Landmarks | Count | Coords per point | Total |
|---|---|---|---|
| Left hand | 21 | x, y, z | 63 |
| Right hand | 21 | x, y, z | 63 |
| Pose | 33 | x, y, z, visibility | 132 |
| **Total** | | | **258** |

Missing landmarks (absent hand) are zero-padded.
