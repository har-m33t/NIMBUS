## What Member 2 built

The full AWS serverless backend that sits between the frontend keypoint stream and the ML model. It handles everything from receiving a WebSocket frame to delivering a spoken English caption.

### Pipeline flow
```
Frontend (keypoints) → WebSocket → ProcessFrame Lambda
  → SageMaker (ASL gloss tokens)
  → DynamoDB (gloss buffer)
  → On sentence boundary → Bedrock/Claude (gloss → English)
  → Polly (English → MP3) → S3 presigned URL
  → WebSocket back → CAPTION + NEW_CAPTION events
```

### Lambda functions deployed
| Function name | Trigger | Purpose |
|---|---|---|
| `NIMBUS_PROD_WS_Connect` | WebSocket `$connect` | Registers connection |
| `NIMBUS_PROD_WS_Disconnect` | WebSocket `$disconnect` | Cleans up connection |
| `NIMBUS_PROD_ProcessFrame` | WebSocket `INFER` action | Main pipeline orchestrator |
| `NIMBUS_PROD_WS_Sweep` | EventBridge every 1 min | Flushes stale gloss buffers |
| `NIMBUS_PROD_WarmEndpoint` | EventBridge every 5 min | Keeps SageMaker warm |

### AWS resources created
| Resource | Name |
|---|---|
| WebSocket API | `NIMBUS_PROD_WS` |
| DynamoDB | `NIMBUS_PROD_Sessions` (TTL 4h, encrypted) |
| DynamoDB | `NIMBUS_PROD_Vocabulary` |
| S3 | `nimbus-prod-tts-audio-899601922590` |
| S3 | `nimbus-prod-config-899601922590` |
| S3 | `nimbus-prod-model-artifacts-899601922590` |
| S3 | `nimbus-prod-session-exports-899601922590` |

### Sentence boundary rules (PROTOCOLS.md §2.2)
The buffer flushes and a CAPTION is emitted when **any** of these fire:
- 15 gloss tokens accumulated
- 3 seconds since first token
- 1.5 seconds without a new frame (handled by Sweep Lambda)
- `[EOS]` token from the model

### Hackathon compliance
- **C1:** Face crops are silently discarded — never stored, never sent to any AWS service. No Rekognition permissions exist in IAM.
- **C2:** Only normalized keypoint floats reach SageMaker — no biometric data.
- **C3:** Bedrock is rate-limited to < 1 RPS via a DynamoDB token bucket.
- **C4:** All S3 buckets have Block Public Access enabled. Audio is served via presigned URLs (5-min TTL).

## For Member 1 — Frontend / Zoom Integration

### WebSocket URL
```
wss://spzn6hu97g.execute-api.us-west-2.amazonaws.com/prod
```
Set this as `NIMBUS_WS_URL` in `frontend/src/main.py`.

### How to connect
Send frames as JSON over the WebSocket with `action: "INFER"`:
```json
{
  "action": "INFER",
  "sessionId": "<client-generated UUID>",
  "roomId": "<zoom-room-id>",
  "timestamp": "2026-04-18T12:00:00.000Z",
  "sequenceNumber": 1,
  "payload": {
    "keypoints": {
      "leftHand":  [{"x": 0.1, "y": 0.2, "z": 0.0}, ...],
      "rightHand": [{"x": 0.5, "y": 0.6, "z": 0.0}, ...],
      "pose":      [{"x": 0.3, "y": 0.4, "z": 0.0, "visibility": 0.99}, ...]
    },
    "includeFaceCrop": false
  }
}
```
- Send at **10 FPS**
- `sessionId` must be generated by the client (UUID), not the server
- All coordinates normalized to `[0.0, 1.0]`

### Events you will receive back
| Event type | When | Key payload fields |
|---|---|---|
| `GLOSS` | Every frame | `tokens`, `confidence` |
| `EMOTION` | Every 10th frame | `emotion: "CALM"` |
| `CAPTION` | On sentence boundary | `text`, `ssmlUrl`, `emotion`, `rawGlossFallback` |
| `SIGNAL` | With every CAPTION | `event: "NEW_CAPTION"`, `text`, `ssmlUrl` |
| `ERROR` | On any failure | `code`, `message` |
| `SIGNAL` | On cold start | `event: "ENDPOINT_WARMING"` |

### Audio playback
`ssmlUrl` in the CAPTION payload is an HTTPS presigned S3 URL pointing to an MP3 file. It expires in **5 minutes**. Play it immediately via the virtual audio device for Zoom.

If `ssmlUrl` is `null`, the model had a Polly failure — display `text` as a text-only caption, no audio.

### Signal quality thresholds (for UI indicator)
| Latency | Status |
|---|---|
| < 800 ms | Green / Strong |
| 800–1500 ms | Yellow / Degraded |
| > 1500 ms | Red / Poor |
| No response in 5s | Gray / Offline |

## For Member 3 — ML Pipeline / SageMaker

### Model artifacts bucket
```
nimbus-prod-model-artifacts-899601922590
```
Upload your trained model here before creating the SageMaker endpoint:
```bash
aws s3 cp model.tar.gz \
  s3://nimbus-prod-model-artifacts-899601922590/models/nimbus-asl/model.tar.gz \
  --region us-west-2
```

### SageMaker endpoint name
The ProcessFrame Lambda is hardwired to call this exact name:
```
nimbus-prod-asl-endpoint
```
Create your endpoint with this name. If you need a different name, let Member 2 know and the `SAGEMAKER_ENDPOINT` env var can be updated via `sam deploy`.

### Input format your endpoint must accept
Content-Type: `application/json`

```json
{
  "instances": [[[f1, f2, ..., f258]]]
}
```
Shape: `(1, T, 258)` where currently `T=1` (single frame). The 258 features are:
- `leftHand`: 21 landmarks × 3 (x, y, z) = **63**
- `rightHand`: 21 landmarks × 3 (x, y, z) = **63**
- `pose`: 33 landmarks × 4 (x, y, z, visibility) = **132**

Missing landmarks are **zero-padded** — you will always receive exactly 258 floats.

### Output format your endpoint must return
```json
{
  "tokens": ["I", "GO", "STORE"],
  "confidence": 0.87
}
```
- `tokens`: list of ASL gloss strings
- `confidence`: float 0–1
- Return `[EOS]` as the last token to signal sentence end

### Inference timeout
The Lambda waits **800ms** for your endpoint before raising an error (falls back to `[UNKNOWN_SIGN]`). Ensure your model responds within that window. If you need more time, ask Member 2 to increase `SAGEMAKER_TIMEOUT_MS`.

### Keeping the endpoint warm
`NIMBUS_PROD_WarmEndpoint` Lambda pings your endpoint every 5 minutes with a zero-padded dummy payload to prevent cold starts. No action needed on your end — just make sure the endpoint handles all-zero input without crashing.
