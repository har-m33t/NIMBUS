HACKATHON CONSTRAINTS (BINDING — OVERRIDE ANY CONFLICTING DETAIL IN PROTOCOLS.md)
---------------------------------------------------------------------------------
C1  Emotion detection IS enabled using Amazon Rekognition DetectFaces.
    - Face crops are accepted only on the defined cadence from INFER payload
      (`includeFaceCrop` true, typically every 10th frame) and used only for
      emotion inference.
    - Face bytes are never persisted to DynamoDB or S3.
    - Emotion outputs are emitted via EMOTION events and used as context for
      Bedrock interpretation and SSML prosody selection.
C2  No personal, biometric, health, financial, political, religious, genetic,
    racial/ethnic, union, payment, regulated, or malicious data may enter AWS.
    - Face crops are processed transiently for emotion classification only and
      are discarded immediately after Rekognition returns.
    - Keypoints (normalized floats) are not biometric identifiers and are allowed.
    - Session recording / export is OFF for this hackathon.
    - No user-provided free-text beyond gloss tokens is forwarded to Bedrock.
C3  Bedrock (generative AI) requests are globally capped at < 1 RPS.
    - Enforced in-Lambda via a token-bucket rate limiter backed by DynamoDB
      atomic counters (see §3.4). Per-session throttle also applied so one
      heavy session cannot starve others.
    - Sentence boundary rules from PROTOCOLS.md §2.2 still apply, but if a
      flush would exceed the global 1 RPS budget, the flush is deferred (up
      to 2 s) or dropped with a raw-gloss fallback CAPTION.
C4  S3 Block Public Access ON for every bucket I create or touch.
    - Bucket-level: BlockPublicAcls, IgnorePublicAcls, BlockPublicPolicy,
      RestrictPublicBuckets ALL = true.
    - Bucket policy explicitly denies `s3:GetObject` to `Principal: "*"`
      unless request is via a presigned URL signed by our role.
    - MP3 delivery is via short-lived presigned URL (ExpiresIn=300s), never
      public-read ACL. No `PublicRead`, no static website hosting.
    - Default SSE-S3 (AES256) encryption enabled on the bucket.

PHASE 1 — INFER INGRESS & SESSION STATE (Days 2–3)
================================================================================

1.1  process_frame handler skeleton
     Route: `INFER` (per PROTOCOLS.md §1.1)
     Steps:
       1. Parse event → pydantic INFER model. Validate:
            - sessionId present (UUID v4)
            - keypoints.leftHand, rightHand arrays (may be empty per §3.3)
            - pose array length 33 or empty
            - sequenceNumber monotonic per session (warn if regressed)
     2. If `includeFaceCrop: true` and `faceCropBase64` is present:
       decode to bytes and run Rekognition emotion detection in-memory.
       Do not persist raw face bytes.
       3. Fetch session state from `NIMBUS_PROD_Sessions` (PK sessionId,
          latest SK by descending timestamp).

1.2  SageMaker invocation (Member 3 contract)
     Client: `boto3.client("sagemaker-runtime")`
     Endpoint: `nimbus-prod-asl-endpoint`
     Input shape: (1, T, 258) per PROTOCOLS.md §7
     T strategy: single-frame per INFER for v1 (T=1); upgrade to rolling
     window after Member 3 confirms model expects temporal context.
     Timeout: 800 ms. On timeout or non-200:
       emit ERROR event with code `SAGEMAKER_INFERENCE_FAILED`,
       glossFallback `[UNKNOWN_SIGN]` (PROTOCOLS.md §4.1).
     Cold start: before first INFER, call DescribeEndpoint; if not InService,
     emit SIGNAL `ENDPOINT_WARMING` (PROTOCOLS.md §4.3).

1.3  Emit GLOSS event immediately on SageMaker response
     Per PROTOCOLS.md §1.2 GLOSS schema. Includes sessionId, sequenceNumber,
     tokens[], confidence. Write to `NIMBUS_PROD_Sessions.glossBuffer`
     (UpdateItem ADD to list).

1.4  Emit EMOTION event from Rekognition output
     On face-crop cadence frames, call `rekognition.detect_faces` and map
     dominant emotion + confidence to EMOTION payload:
       { type: "EMOTION", payload: { emotion: "HAPPY|...", confidence: n,
         allEmotions: {...} } }
     On missing/invalid face crop or Rekognition failure, fallback to CALM
     (confidence 1.0) to preserve contract and availability.

PHASE 2 — GLOSS BUFFER & SENTENCE BOUNDARY (Days 3–4)
================================================================================

2.1  Buffer location
     `NIMBUS_PROD_Sessions` item per PROTOCOLS.md §2.1:
        PK=sessionId, SK=timestamp, glossBuffer list<string>,
        lastEmotion "CALM", lastCaptionAt, connectionId, ttl (now+4h).

2.2  Boundary detector (PROTOCOLS.md §2.2 — all four rules)
     Flush conditions (OR):
       a. len(glossBuffer) >= 15
       b. now() - firstTokenAt >= 3.0 s
       c. now() - lastTokenAt >= 1.5 s  (pause signal)
       d. any token == "[EOS]"
     Rule (c) cannot be triggered from inside an INFER handler alone — add a
     scheduled sweeper: EventBridge rule every 1 s invoking
     `NIMBUS_PROD_ProcessFrame` with action `SWEEP` that scans sessions whose
     `lastTokenAt` is stale and flushes them.

2.3  Flush procedure
     - Conditional UpdateItem to atomically drain glossBuffer and stamp
       lastCaptionAt (prevents double-flush races with sweeper).
     - Read last 3 captions for this sessionId (GSI on sessionId sorted by
       lastCaptionAt, LIMIT 3).
     - Call bedrock_interpret(gloss_tokens, emotion="CALM", context=[...]).
     - Call polly_synthesize(english, emotion="CALM").
     - Emit CAPTION event per PROTOCOLS.md §1.2 (to originating session).
     - Emit SIGNAL `NEW_CAPTION` per §1.2 with {roomId, caption payload}
       so Member 1's `NIMBUS_PROD_BroadcastCaption` fans out to all room
       participants.
     - Write CAPTION payload back into DynamoDB as a new item for context.

PHASE 3 — BEDROCK INTERPRETER (Days 4–6)
================================================================================

3.1  Service: services/bedrock_interpreter.py
     Client: `boto3.client("bedrock-runtime")`
     API: `converse`
     Model: `anthropic.claude-haiku-4-5` (env BEDROCK_MODEL_ID)

3.2  Prompt (system block — prompt-cached)
     You are an expert ASL-to-English interpreter. Convert ASL gloss token
     sequences into fluent, grammatically correct English. ASL omits articles,
     uses topic-comment order, and marks tense with temporal tokens
     (YESTERDAY, FINISH, WILL). Rules:
       - Preserve meaning; do not invent facts.
       - Output ONE sentence unless gloss clearly contains multiple.
       - Return strict JSON: {"english": "...", "confidence": 0.0-1.0}.
       - If gloss is too short/ambiguous:
         {"english": null, "reason": "insufficient"}.

     User block:
       <prior_captions>{up to 3 previous English sentences}</prior_captions>
       <emotion>CALM</emotion>
       <gloss>YESTERDAY STORE I GO FINISH MILK BUY</gloss>

3.3  Inference config
     maxTokens=120, temperature=0.2, topP=0.9
     Timeout: 1500 ms. On timeout emit ERROR `BEDROCK_TIMEOUT`, caption
     falls back to space-joined gloss (PROTOCOLS.md §4.1).

3.4  RATE LIMITING (C3 — strict < 1 RPS global)
     Token bucket in DynamoDB:
       Table: `NIMBUS_PROD_Sessions` (reuse) or new `NIMBUS_PROD_RateLimit`
       Item: PK="bedrock_global"
       Attrs: tokens (Number), lastRefillMs (Number)
       Config: capacity=1, refillRate=1/sec (one token per second)

     Before each invoke:
       1. UpdateItem with ConditionExpression:
            "tokens >= :one"
          Atomic decrement + refill based on now() - lastRefillMs.
       2. If ConditionalCheckFailed:
            - Retry up to 2 s with 100 ms backoff.
            - If still blocked: emit ERROR `BEDROCK_TIMEOUT` with raw-gloss
              fallback caption. Metric: BedrockRateLimited++.

     Per-session soft cap: max 1 Bedrock call every 2 s per sessionId,
     tracked on the session item itself, to prevent one session monopolizing.

     Additional defensive layer: request a Bedrock service quota INCREASE
     to zero is not possible, so we rely on our in-Lambda limiter. Also set
     Lambda reserved concurrency on `NIMBUS_PROD_ProcessFrame` = 10 to
     cap maximum theoretical RPS.

3.5  Robustness
     - JSON parse retry once with "Return ONLY valid JSON." appended.
     - Circuit breaker: 3 consecutive Bedrock errors within 30 s → open
       circuit for 60 s; all flushes use raw-gloss fallback during open state.
     - Cost guard: if gloss tokens >50, truncate to last 50 (should be
       unreachable given boundary rule a = 15 tokens).

3.6  Model quality check (Day 12)
     Run 50-sample gloss fixture through Haiku vs Sonnet; pick the cheaper
     one if quality delta is small. Doc in PRs.

5.4  Synthesize flow
     def synthesize(text: str, emotion: str = "CALM") -> AudioArtifact:
         ssml = build_ssml(text, emotion)
         audio = polly.synthesize_speech(Text=ssml, TextType="ssml",
                    OutputFormat="mp3", VoiceId=voice, Engine="neural")
         key = f"{sessionId}/{uuid4()}.mp3"
         s3.put_object(Bucket="nimbus-prod-tts-audio", Key=key,
                       Body=audio["AudioStream"].read(),
                       ContentType="audio/mpeg",
                       ServerSideEncryption="AES256")
         url = s3.generate_presigned_url("get_object",
                 Params={"Bucket":"nimbus-prod-tts-audio","Key":key},
                 ExpiresIn=300)
         return AudioArtifact(url=url, key=key)

     CAPTION event includes `audioUrl` (the presigned URL). Note
     PROTOCOLS.md §1.2 shows `s3://` — I will emit the HTTPS presigned URL
     and raise this as a spec deviation to confirm with Member 1 (Q3).

5.5  Polly failure fallback (PROTOCOLS.md §4.1)
