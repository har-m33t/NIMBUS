## Project Overview

NIMBUS is a real-time ASL (American Sign Language) interpretation system that integrates with Zoom. It captures webcam keypoints via MediaPipe, streams them to an AWS serverless backend, runs ASL inference (SageMaker Transformer), translates gloss tokens to English (Amazon Bedrock/Claude), detects emotion (Amazon Rekognition), generates expressive speech (Amazon Polly), and delivers captions + audio into Zoom meetings.

### Sentence boundary conditions (PROTOCOLS.md Â§gloss-buffer)

A gloss buffer flush is triggered by **any** of:
- Buffer reaches 15 tokens
- 3 seconds elapsed since first token
- 1.5 seconds without a new frame
- `[EOS]` token detected

### Key invariants

- Keypoints are sent at **exactly 10 FPS**; face crop JPEG (max 640Ă—480, base64) every 10th frame
- All coordinates normalized to `[0.0, 1.0]` (MediaPipe convention)
- Session IDs are generated by the **client**, not the server
- SSML prosody is read from `nimbus-prod-config` S3 at cold start (edit `ssml_prosody_map.json` in S3 to retune without redeployment)
- `NIMBUS_PROD_WarmEndpoint` Lambda runs on a schedule to prevent 30â€“90s SageMaker cold starts
- Every service failure must emit an `ERROR` WebSocket event with a defined fallback (never crash silently): SageMaker â†’ `[UNKNOWN_SIGN]`, Bedrock â†’ raw gloss, Rekognition â†’ `CALM`, Polly â†’ caption without audio
